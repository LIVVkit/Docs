Advanced installation¶

Latest release¶

The latest LIVVkit release can be installed via Python pip:

pip isntall --user livvkit

Via Anaconda/Miniconda:

conda install -c jhkennedy livvkit

Or by cloning the source from:

git clone https://github.com/LIVVkit/LIVVkit.git

LIVVkit was designed to be Python 2 and 3 compatible, so either version will work, but Python 3 is recommended. If you’re having problems installing LIVVkit, see our detailed Advanced installation page, or open an issue on github.

For development¶

LIVVkit uses a public-private development model. The main development happens in a private repository under the Department of Energy’s ACME project on GitHub. The LIVVkit codebase is periodically released to public the LIVVkit project on GitHub, and provided as a package on PyPI and Anaconda Cloud.

There are two encouraged ways to contribute to the development of LIVVkit:

1. For ACME developers¶

First, contact Joseph H. Kennedy and ask to be added to the ACME-Climate/LIVV repository. Once you have access to the repository, clone the code and checkout the development branch:

git clone https://github.com/ACME-Climate/LIVV.git
git checkout develop

When developing the code, please follow the GitFlow workflow and make all your changes in a feature branch. Feature branches should follow the username/feature naming convention.

If you have any questions, concerns, requests, etc., open an issue in our private issues queue, and we will help you out.

2. For non-ACME developers¶

Please use the Forking Workflow to add contributions to LIVVkit.

First, go to the LIVVkit github page and push the Fork button on the top right of the page. This will create a fork of LIVVkit in your profile page. Clone the fork, make your changes, merge them to development branch, and then submit a pull request to our public repository.

If you have any questions, concerns, requests, etc., open an issue in our public issues queue, and we will help you out.
